# 開発実行計画：高機能タスク管理システム

「設計・開発・テスト・デプロイ」の全工程を、実務の作法に則って完遂する計画

## 1. 開発フェーズとスケジュール（目安）

### 第1フェーズ：上流工程（設計） 【期間：約1〜2週間】
実務の「なぜこの設計にしたか」を説明できる根拠を作ります。
- [X] **要件定義書の作成**: 機能・非機能要件の確定 (`docs/phase1-design/requirements.md`)
- [X] **DB設計（ER図）**: テーブル定義とリレーションシップの構築 (`docs/phase1-design/db-design.md`)
- [X] **API設計**: エンドポイント、リクエスト/レスポンス形式の定義 (`docs/phase1-design/basic-design.md`)
- [X] **インフラ構成設計**: AWSの利用サービス選定と構成図作成 (`docs/phase2-setup/infrastructure.md`)

### 第2フェーズ：詳細設計・環境構築 【期間：約1週間】
実装をスムーズにするための土台作りです。
- [X] **コンテナ環境構築**: `docker-compose` によるJava/DB実行環境の構築
- [X] **プロジェクト作成**: Spring Initializrでのスケルトン作成
- [X] **共通設計**: 例外ハンドリング方針、ログ出力方針の決定

**学習ログ**:
- [詳細設計・環境構築](docs/phase2-setup/infrastructure.md)

### 第3フェーズ：実装（メイン開発） 【期間：約4〜6週間】
「保守性の高いコード」を意識したコーディングを行います。

- [ ] **Vol.1 認証基盤**: Spring Security + JWTによるログイン機能
- [X] **Vol.2 タスクCRUD**: 基本的なタスク管理機能の実装
- [ ] **Vol.3 検索・抽出**: 効率的なクエリによる検索機能
- [ ] **Vol.4 バッチ/通知**: 非同期処理、またはスケジュール実行による通知機能
- [ ] **Vol.5 共通品質**: バリデーションの徹底とリファクタリング

**詳細な機能仕様は [こちら](docs/phase1-design/requirements.md) を参照**

**学習ログ**:
- [Vol.1 認証基盤](docs/phase3-impl/vol1_auth.md)
- [Vol.2 タスクCRUD基礎](docs/phase3-impl/vol2_task_crud.md)

### 第4フェーズ：テスト・品質保証 【期間：約2週間】
市場価値を左右する「テストコード」の充実を図ります。
- [ ] **単体テスト**: JUnit 5 / Mockitoを用いたService層の網羅（カバレッジ80%以上）
- [ ] **結合テスト**: APIエンドポイントに対する疎通・振る舞いテスト
- [ ] **静的解析**: CheckstyleやSpotBugs等によるコード品質チェック

### 第5フェーズ：デプロイ・CI/CD 【期間：約1週間】
モダンなインフラ運用経験を積みます。
- [ ] **AWSデプロイ**: App Runner, RDSを用いた本番公開
- [ ] **CI/CD構築**: GitHub Actionsによるテスト・デプロイの自動化

## 2. 実務を想定した運用ルール（個人開発の制約）

- **Gitワークフロー**:
    - `main` ブランチへ直接プッシュ禁止。
    - `feature/issue-番号` でブランチを作成し、セルフプルリクエストを経てマージする。
- **ドキュメント管理**:
    - 全ての設計書は `docs/` フォルダ内でMarkdown形式で管理する。
    - 変更履歴をコミットメッセージに残す。
- **コード品質**:
    - 常に「他人が読む」ことを意識し、JavaDocや適切な変数名を心がける。
