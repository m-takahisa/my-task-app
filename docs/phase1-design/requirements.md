# 要件定義書：高機能タスク管理システム

## 1. プロジェクト概要
### 1.1 背景と目的
- **現状の課題**: 現在の現場環境では望んでいるスキルの向上が見込めない(Javaのバージョンが古い、コンテナ・クラウドに携わる機会がほぼない)。上流工程を経験する機会がほぼない。
- **解決策**: Spring Bootとモダンな技術スタックを用いた、堅牢なタスク管理システムを自作する。
- **目標**: 30代Javaエンジニアとしての技術力証明（Spring Boot 3, AWS, CI/CD）および、実務レベルの設計・開発プロセスの完遂。

## 2. ターゲットユーザー
- シンプルなタスク管理を求める個人ユーザー。
- Webブラウザから場所を問わずアクセスしたいユーザー。

## 3. 機能要件

### 3.1 ユーザー管理機能 (Vol.1)
- **ユーザー登録**: メールアドレス、パスワード、名前による新規登録。
- **ログイン/ログアウト**: JWT（JSON Web Token）を用いたセキュアな認証。
- **認可制御**: ログインユーザーのみが自身のデータにアクセス可能とする（他人のタスクは操作不可）。

### 3.2 タスク管理機能 (Vol.2)
- **タスク操作**: タイトル(100字以内)、内容、期限、ステータスの登録・編集・削除が可能。
- **バリデーション**: `@NotBlank` による必須チェック、`@Size` による文字数制限を実装済み。
- **一覧表示**: 全件取得による一覧表示（※Vol.4以降でユーザー毎に切り替え）。

### 3.3 検索・フィルタリング機能 (Vol.3)
- **キーワード検索**: タイトルおよび内容からの `LIKE` 検索。
- **絞り込み**: ステータス、期限（本日・今週など）による抽出。

### 3.4 通知・バッチ機能 (Vol.4)
- **期限リマインド**: 期限が当日または翌日のタスクを自動抽出し、通知（ログ出力やメール送信）を行う。

### 3.5 共通仕様・バリデーション (Vol.5)
- **入力チェック**: 文字数制限、必須入力、未来日の日付制限など。
- **共通エラーハンドリング**: 全てのAPIで統一されたエラーレスポンス形式（JSON）の返却。

## 4. 非機能要件（品質・運用）
- **セキュリティ**: パスワードのBCryptハッシュ化、CORS対策。
- **保守性**: 3層アーキテクチャの採用、JUnit 5によるユニットテスト。
- **可用性**: AWS（App Runner / RDS）へのデプロイ。
- **CI/CD**: GitHub Actionsによるビルド・テスト・デプロイの自動化。

## 5. 技術スタック
- **Backend**: Java 21 / Spring Boot 3.x
- **Database**: PostgreSQL
- **Infrastructure**: Docker / AWS
- **Build**: Gradle or Maven
